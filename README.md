# 賀卡影片產生器（純前端）

把「名字＋賀語」文字燒進賀卡短影片的純前端網頁工具。無後端，可放 GitHub Pages 或公司內網。
所有處理都在使用者瀏覽器內用 **ffmpeg.wasm** 完成，影片不會上傳。

- `index.html` — 專員產出頁
- `editor.html` — 管理者範本編輯頁
- `templates.json` — 當月所有範本設定（管理者每月更新這一份）

---

## 一、管理者指南

### 1. 建立/更新範本（每月一次）
1. 開 `editor.html`。
2. **上傳影片**（只用於定位、讀解析度與檔名，會自動擷取一張縮圖）。影片實體請另外上傳到你們的 Google Drive 共用資料夾。
3. 按「＋名字欄位 / ＋賀語欄位」新增文字框，直接在預覽上**拖曳定位**；右側面板設定：欄位類型、預設文字、字體、字級、粗細、對齊、**文字色／描邊色／陰影色**（色票＋自訂色碼）。
4. 填「範本名稱、說明、Google Drive 分享連結」。
5. 按「儲存到清單」。要多個範本就「新增另一個範本」重複。
6. 全部設定好後按「**匯出 templates.json**」，把檔案放回網站根目錄覆蓋舊的即完成。
   - 要延續上月內容：先「匯入 templates.json」再增修。

### 2. Google Drive 分享設定（重要）
- 每支影片在 Drive 上按右鍵 → 共用 → 一般存取權改「**知道連結的任何人**」→ **檢視者**。
- 編輯頁有「產生直接下載連結」按鈕，會把分享連結轉成 `uc?export=download` 形式。
- ⚠️ 檔案 **大於約 100MB** 時，Google 會先跳「無法掃描病毒」的確認頁，專員需再點一次「仍要下載」——這是 Google 的限制，無法完全跳過。建議範本影片盡量小。

### 3. 部署到 GitHub Pages
1. 把整個資料夾推到 GitHub repo。
2. repo → Settings → Pages → Source 選 `main` 分支 `/ (root)` → Save。
3. 幾十秒後可用 `https://<帳號>.github.io/<repo>/` 開啟。
4. 每月更新只要替換 `templates.json`（和新縮圖，如果縮圖用檔案而非 base64）再 push 即可。
- 公司內網：把整個資料夾丟到任何靜態網頁伺服器即可（本工具部署為公開版，字體走 Google Fonts CDN；若要純離線，需另外把字體與 ffmpeg 檔案改為站內託管）。

## 二、專員指南（四步驟）

1. **選範本**：首頁點當月要用的賀卡卡片。
2. **下載影片**：按「前往 Google Drive 下載影片」（新分頁），下載到電腦。
   （若跳出 Google 的「下載確認」頁，點確認即可。）
3. **拖入影片**：把剛下載的影片檔拖到步驟 2 的框框。
   - 系統會用檔名比對，拖錯會提示「這不是本範本的影片」。
   - 拖對後上方會出現預覽。
4. **輸入文字 → 產出**：
   - 填名字（必填）、賀語（可改）；旁邊小色塊可改文字顏色；上方預覽即時顯示。
   - 按「產出賀卡影片」，等進度條跑完會自動下載，檔名為 `範本名稱_名字.mp4`。
   - 要換名字再做一支：直接改文字再按產出，**不用重新拖影片**。

---

## 三、已知限制

- **處理速度**：採單執行緒 ffmpeg.wasm、整支影片重新編碼。1080p、數十秒的短片，通常每支約 **15～60 秒**（視電腦而定）；影片越長越大越慢。
- **建議規格上限**：1080p、≤60 秒、≤200MB。超過可能很慢或記憶體不足；拖入過大檔案會提醒。
- **為何要手動下載再拖入**：瀏覽器 CORS 安全限制，無法直接從 Google Drive 抓取影片內容，所以設計為「開連結下載 → 拖入」，請勿期待自動抓取。
- **瀏覽器**：請用 **Chrome 或 Edge**。Safari／Firefox 對 ffmpeg.wasm 支援不穩，可能無法產出。
- **音訊**：若原影片有音訊會原樣保留；沒有就純影像輸出。
- **字體**：使用 Google Fonts（簡體中文含 Noto Sans SC／Noto Serif SC 等 5 種）。燒錄前會強制載入該段文字所需字形，確保輸出與預覽一致。需要能連到 Google Fonts；純離線環境請改為站內託管字體。

## 技術重點
- 文字用 **Canvas 渲染成透明 PNG 圖層**（尺寸＝影片原始解析度，位置/字級/描邊/陰影皆以「相對比例」換算回原始解析度），再用 ffmpeg.wasm 的 **overlay** filter 疊上去，確保 WYSIWYG；不使用 `drawtext`（中文支援差）。
- ffmpeg.wasm 用 **單執行緒 core**，不需 SharedArrayBuffer，因此不需要 COOP/COEP header，GitHub Pages 可直接跑。
- 同一支影片只寫入虛擬檔案系統一次，換名字時只更換 PNG 圖層重跑，並釋放中間檔案。
